# PAH Visualizer: Development Progress Log

## Recent Progress & Technical Updates

### 1. Feature-Sliced Architecture Refactor
The application underwent a massive architectural overhaul to improve maintainability and scalability. The codebase was restructured from a flat src/ directory into a feature-sliced design:
- **src/features/**: Contains domain-specific logic (3d, calculator, devkit). Experience.jsx and its subcomponents were moved here.
- **src/layouts/**: Houses structural components like LeftSidebar.jsx and PresetPanel.jsx to clean up the App.jsx root.
- **src/components/**: Retained for generic, reusable UI elements.
- **src/stores/**: Renamed from store/ for convention.

### 2. Single-Canvas 3D Layout Unification
Addressed a critical performance issue where App.jsx previously mounted two separate <Experience /> WebGL contexts𤤖ne for mobile and one for desktop𤪔iding the inactive one via CSS (display: none). 
- **The Fix**: The layout was unified using responsive CSS Grid/Flexbox on a single DOM tree. The <Experience /> component is now mounted exactly once, dynamically resizing and repositioning itself depending on the viewport (side-by-side flex on desktop, vertical stack on mobile), saving massive amounts of memory and preventing mobile crashes.

### 3. Zustand useShallow Mocking for Tests
Unit tests (App.test.jsx, Experience.test.jsx) were updated to correctly mock Zustand stores that utilize the useShallow hook for performance optimizations. The mock pattern was evolved from a simple object return to a selector-function pattern (i.fn((selector) => selector ? selector(state) : state)), allowing the tests to accurately simulate the state selection process without triggering "Cannot read properties of undefined" errors.

### 4. High-Fidelity Asset Pipeline Optimization
To ensure maximum accuracy in our procedural generation, we overhauled the texture compression pipeline.
- **Removed 512px Constraints:** The previous pipeline applied a 512px downscaling step which degraded the UV mask accuracy. We removed this constraint to prioritize fidelity for raycasting.
- **Original Resolution JPEGs:** We transitioned from PNGs to high-quality .jpeg (Quality 0.95) equivalents for scalp_mask.jpeg and scalp_uv_guide.jpeg. This provides the optimal balance of web performance while maintaining pixel-perfect parting lines and region boundaries (Top, Sides, Back) for the raycaster.

### 5. Architecture Discovery & Pipeline Mapping
A comprehensive review of the core rendering components was conducted to document the pipeline from math to pixels.

#### Experience.jsx - The Orchestrator
- Functions as the primary wrapper for the R3F Canvas.
- Manages the environment (lighting, shadows, camera controls).
- Integrates the HeadModel (the scalp canvas) and HairStrands (the generated braids).

#### usePartingPattern.js - The CPU Engine
- Uses spherical raycasting to project points onto the head mesh.
- Intersects with scalp_mask.jpeg to identify specific spawning regions based on RGB channels (Top=Red, Sides=Green, Back=Blue) and enforces parting gaps (Black).
- Ensures hairs are only placed on valid scalp geometry, avoiding the face and neck.

#### HairStrands.jsx - The GPU Renderer
- Takes the coordinate data generated by the CPU and uses InstancedMesh to render tens of thousands of braid segments efficiently.
- Implements procedural physics tracing, using gravity-biased direction vectors and ellipsoidal collision boundaries (head/torso spheres) to simulate natural hair draping.
- Applies custom shaders for Fresnel sheen to simulate realistic hair highlights.
